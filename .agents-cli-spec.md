# Agent Spec: Clinical Trial Agent

## Overview
An ADK (Agent Development Kit) agent designed to search, retrieve, and synthesize clinical trial information. It helps researchers, clinicians, and patients locate ongoing or completed clinical studies and understand their status, sponsor, and details.

## Example Use Cases
### Case 1: Search for Type 2 Diabetes Trials
* **Input:** "Find clinical trials for type 2 diabetes"
* **Expected Output:** A synthesized summary of matching clinical trials, including trial titles, phases, status, and sponsors, sourced from ClinicalTrials.gov and Google Search.

## Tools Required
1. **ClinicalTrials.gov Tool:** Search condition/disease name to return official trial records (title, phase, status, sponsor).
2. **Google Search Grounding:** To search for additional context or news regarding specific clinical trials.

## Constraints & Safety Rules
* **No Medical Advice:** The agent must explicitly state that the information provided is for research purposes only and does not constitute medical advice.
* **Source Attribution:** Always cite where the trial data was obtained (e.g., ClinicalTrials.gov API, Google Search, or Local Fallback).
* **Factual Grounding:** Rely strictly on verified information and avoid hallucinating trial outcomes or details.

## Success Criteria
* Correctly query and extract clinical trial listings.
* Successfully fall back to a local mock database when offline or if the external API is down.
* Correctly integrate ADK agent primitives (`Agent`, `Tool`, `Runner`, etc.).
